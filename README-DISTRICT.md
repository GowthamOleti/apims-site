# District Design Hub

A clean, Apple-inspired internal tool for design resources. Built as a single static HTML file (`design-hub-live.html`) that talks directly to Supabase (Postgres + Auth + Storage). Google Sign‑In via Supabase Auth. No build step required.

## Features
- Google Sign‑In (Supabase Auth)
- Profiles stored in `profiles` table
- Resources stored in `resources` table (type, title, description, url, tags, author, created_at)
- Category filtering (UI Findings, Podcasts, Books, Articles, Videos, Tools, etc.)
- Search across title/description/tags/author
- Empty-state handling and toasts
- Light, monochrome Apple-like UI

## Quick start
1. Serve the static file:
   - Python 3: `python3 -m http.server 3000`
   - Or any static server: `npx serve`
2. Visit `http://localhost:3000/design-hub-live.html`.

## Configure Supabase
1. In `design-hub-live.html`, set your Supabase credentials:
   ```js
   const SUPABASE_URL = 'https://YOUR-PROJECT-id.supabase.co';
   const SUPABASE_ANON_KEY = 'YOUR-ANON-PUBLIC-KEY';
   ```
2. Supabase Auth > Settings:
   - Site URL: `http://localhost:3000`
   - Additional Redirect URLs: `http://localhost:3000/design-hub-live.html`
   - Enable Google provider

## Database schema (Supabase SQL)
```sql
-- profiles
create table if not exists profiles (
  id uuid primary key references auth.users(id) on delete cascade,
  name text,
  email text unique,
  avatar_url text,
  created_at timestamp with time zone default now()
);

-- resources
create table if not exists resources (
  id bigint generated by default as identity primary key,
  type text not null,         -- ui-finding | podcast | book | article | video | tool
  title text not null,
  description text,
  url text,
  tags text[] default '{}',
  author_name text not null,
  created_at timestamp with time zone default now(),
  created_by uuid references auth.users(id)
);

alter table profiles enable row level security;
alter table resources enable row level security;

create policy "Public read profiles" on profiles for select using (true);
create policy "Users manage own profile" on profiles for
  insert with check (auth.uid() = id),
  update using (auth.uid() = id);

create policy "Public read resources" on resources for select using (true);
create policy "Authenticated can insert resources" on resources for insert with check (auth.uid() is not null);
```
